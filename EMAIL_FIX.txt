# 📧 EMAIL NOT WORKING? - QUICK FIX

## 🚨 Most Common Issue: App Password Not Set

### ✅ 3-Step Fix (5 Minutes)

---

## STEP 1: Get Gmail App Password

1. **Enable 2-Step Verification** (REQUIRED!)
   - Go to: https://myaccount.google.com/security
   - Turn ON "2-Step Verification"
   - Follow the setup

2. **Generate App Password**
   - Go to: https://myaccount.google.com/apppasswords
   - Select: Mail → Other (Custom name)
   - Name it: AyurAI Veda
   - Click Generate
   - **COPY THE 16-CHARACTER PASSWORD**
   - Example: `abcdefghijklmnop` (NO SPACES!)

---

## STEP 2: Add to Vercel

1. Go to: https://vercel.com/dashboard
2. Select your project
3. Settings → Environment Variables
4. Add these 3 variables:

```
Name: SENDER_EMAIL
Value: zjay5398@gmail.com
Environment: ✅ Production ✅ Preview ✅ Development

Name: GMAIL_APP_PASSWORD  
Value: your_16_char_password (NO SPACES!)
Environment: ✅ Production ✅ Preview ✅ Development

Name: ADMIN_EMAIL
Value: zjay5398@gmail.com
Environment: ✅ Production ✅ Preview ✅ Development
```

5. Click Save for each

---

## STEP 3: Redeploy

### Option A: Via Dashboard
1. Go to Deployments tab
2. Click "..." on latest deployment
3. Click "Redeploy"

### Option B: Via Git
```bash
git commit --allow-empty -m "Redeploy for email config"
git push
```

### Option C: Via CLI
```bash
vercel --prod --force
```

---

## ❌ Common Mistakes

1. **Using Gmail password instead of App Password**
   - ❌ Your regular Gmail password
   - ✅ 16-character App Password

2. **App Password has spaces**
   - ❌ `abcd efgh ijkl mnop`
   - ✅ `abcdefghijklmnop`

3. **2-Step Verification not enabled**
   - Must be ON to generate App Passwords

4. **Environment variables not set for all environments**
   - Must check: Production, Preview, Development

5. **Forgot to redeploy after adding variables**
   - Variables only work after redeploy
